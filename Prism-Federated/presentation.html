Prism-Federated

Privacy-preserving distributed intelligence for sensitive data silos. Train a global model without centralizing raw data — only model weights move over gRPC. Runnable FedAvg-on-MNIST lab with Flower + PyTorch.

FlowerPyTorchFedAvggRPC
01
The Problem

The hurdle isn't the algorithm — it's data access

Under GDPR/HIPAA, sensitive workloads (e.g. medical imagery) can't leave each site's firewall. Prism-Federated trains a shared global model across simulated hospital sites where no raw patient data crosses the wire — only model updates flow to a central aggregator.

Topology

Weights move, data doesn't

Hospital A --local training--\ Hospital B --local training---+--( protected model updates only )--> [ Aggregator ] Hospital C --local training--/ FedAvg + global model (raw records never leave each firewall) Why gRPC over Protobuf: compact binary framing for large float-tensor weight transfer vs text-heavy REST+JSON re-serialization.
