01
The Problem

The cold-start feature problem

Most fraud models are built for batch. Sentinel-Stream is built for the moment of transaction: how do you know a user made five purchases in the last ten minutes if your database hasn't updated yet? A Kappa architecture processes, aggregates, and scores in ~60ms on the inference hop — validate against your own SLO and hardware.

02
Key Features

Latency-safe by design

Shadow mode

Challenger model scores every event (shadow_fraud_probability); production decision stays on the champion until you promote.

Circuit breaker

If ML inference exceeds the threshold (default 60ms), the API falls back to a high-speed heuristic so the payment path never blocks. State reported per response.

Stateful windows

Apache Flink (target) maintains a living user profile — velocity, geo-divergence, session aggregates — updated every event. Phase 0 simulates with Parquet.
